## 2024-05-24 - Accessibility in Frameless PyQt6 Windows
**Learning:** When using frameless custom windows in PyQt6 (by setting `Qt.WindowType.FramelessWindowHint`), the default OS-level window decorations and accessibility features are completely lost. This means icon-only control buttons (like minimize and close) built from scratch lack both visual tooltips and screen-reader context (ARIA equivalent for desktop apps) by default, creating a severe accessibility barrier for users relying on keyboard navigation or screen readers. Similarly, `QSystemTrayIcon` objects can be opaque to screen readers if they lack a descriptive tooltip.
**Action:** Always explicitly implement `setToolTip()` for visual hints and `setAccessibleName()` for screen reader context when creating custom window control buttons or tray icons in frameless PyQt6 applications.
