## 2026-04-26 - [Clipboard Sniffing Exposure]
**Vulnerability:** The password was being unconditionally copied to the system clipboard in `on_item_clicked` within `secure_switcher.py`, even when the "Omni Login" feature (which simulates keystrokes directly) was active, effectively defeating its purpose and leaving credentials exposed in the clipboard indefinitely.
**Learning:** Security features meant to bypass vulnerable channels (like the clipboard) must explicitly verify that those channels aren't being inadvertently utilized in parallel code paths. Always ensure features meant to prevent data leaks do not unintentionally log or copy the data.
**Prevention:** For OmniVault's "Omni Login" feature, the code should branch: if `riot_logic` is active, completely bypass clipboard operations. If normal clipboard copying occurs, always implement an automatic timeout (e.g., using `QTimer.singleShot(30000, ...)`) to clear the sensitive data from the clipboard after a reasonable period to minimize exposure time to clipboard-sniffing malware.
