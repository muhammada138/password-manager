## 2024-05-24 - Clipboard Bypass Leak & Exposure
**Vulnerability:** The application was unconditionally copying sensitive passwords to the system clipboard upon clicking an entry, even when the user had explicitly enabled "Omni Login" (a feature specifically designed to *bypass* the clipboard to prevent clipboard-sniffing attacks by typing the password instead). Furthermore, passwords copied to the clipboard normally were never cleared, leading to indefinite exposure.
**Learning:** This highlights a critical oversight in the implementation of "Omni Login". While the feature correctly typed out the password, it failed to suppress the default copy action. Additionally, standard clipboard operations for sensitive data must always be temporary to reduce the attack surface.
**Prevention:** Always verify that feature toggles designed for security (like "Omni Login") completely disable the less secure fallback behavior. Implement automatic timeouts (e.g., 30 seconds) for sensitive data stored in the clipboard, and clear it conditionally only if it hasn't been overwritten by the user in the meantime.
